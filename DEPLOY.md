# Deploy Learnograph to EdgeOne Pages

This document describes how to deploy Learnograph on [EdgeOne Makers (pages.edgeone.ai)](https://edgeone.ai/makers), the hackathon's sponsored platform.

## Prerequisites

- A GitHub account with this repo pushed
- An EdgeOne Makers account at [edgeone.ai/makers](https://edgeone.ai/makers)
- An AI Gateway API key (from EdgeOne Makers Models)

## Step-by-Step Deployment

### 1. Get an AI Gateway API Key

1. Go to [EdgeOne Makers Console](https://edgeone.ai/makers)
2. Navigate to **Models → API Key**
3. Click **Create Key** and copy it — you'll need it in step 3

> The built-in `@makers/deepseek-v4-flash` model is free with a usage cap and is used by default.

### 2. Connect Your Repository

1. Go to [EdgeOne Makers → New Project](https://edgeone.ai/makers/new)
2. Select **Import from GitHub** and connect your repository
3. EdgeOne will auto-detect the framework (Vite/React)

### 3. Configure Build Settings

EdgeOne should auto-detect these from `edgeone.json`, but verify:

| Setting           | Value           |
|-------------------|-----------------|
| Build Command     | `npm run build` |
| Output Directory  | `dist`          |
| Framework         | Auto (Vite)     |

### 4. Set Environment Variables

In the project's **Settings → Environment Variables**, add:

| Variable              | Value                                          | Required |
|-----------------------|------------------------------------------------|----------|
| `AI_GATEWAY_API_KEY`  | Your Makers API key from step 1                | ✅ Yes   |
| `AI_GATEWAY_BASE_URL` | `https://ai-gateway.edgeone.link/v1`           | ✅ Yes   |
| `AI_GATEWAY_MODEL`    | `@makers/deepseek-v4-flash` (or your choice)   | Optional |

> **Alternative**: If you prefer to use the Anthropic API directly, set `ANTHROPIC_API_KEY` instead and leave the gateway vars empty.

### 5. Deploy

Click **Deploy** — EdgeOne will:
1. Install dependencies (`npm install`)
2. Build the frontend (`npm run build`)
3. Deploy the `dist/` directory as static files
4. Deploy `cloud-functions/` as serverless functions
5. Deploy `agents/` as agent functions

Your app will be live at the generated EdgeOne URL.

### 6. Verify

1. Open the deployed URL
2. Type a topic (e.g., "React Hooks") and click **Generate Roadmap**
3. Once the tree appears, paste a relevant URL and click analyze
4. Confirm/dismiss the AI suggestion and watch the progress bar update

## Project Structure

```
learnographer/
├── dist/                          # Built frontend (generated by Vite)
├── cloud-functions/               # Serverless API endpoints
│   ├── generate-tree/index.py    # POST /generate-tree
│   ├── analyze-url/index.py      # POST /analyze-url
│   ├── history/index.py          # POST /history (template)
│   └── ...
├── agents/                        # EdgeOne Agent functions (template)
├── src/                           # React frontend source
├── edgeone.json                   # EdgeOne deployment config
└── package.json
```

## Troubleshooting

- **"AI Gateway API key is not configured"**: Make sure `AI_GATEWAY_API_KEY` is set in EdgeOne project settings
- **"Failed to connect to AI Gateway"**: Verify `AI_GATEWAY_BASE_URL` is correct
- **Build fails**: Run `npm run build` locally to check for TypeScript errors
